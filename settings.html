🔧 Backfill / Maintenance
For a story that already has summaries: the auditor and ledger only run on new batches, so existing snippets have no detail notes and the ledger starts empty. These replay the existing history to catch them up. Both run in the background, one item at a time, and can be stopped anytime (click the ✕ on the progress toast) — progress is saved. For a single snippet instead, use the 🪄 (detail) and 🧠 (ledger) buttons in the Snippet Browser.
Build Ledger replays the whole story in Turns per Summary Batch-sized passes and merges into the current ledger (Clear Ledger first for a clean rebuild). Backfill Missing Details audits only snippets that don't have a note yet — existing notes are untouched. Also available as /sc-ledger-build and /sc-audit-all.

✏️ Summarizer Prompts
🔍 Detail Auditor (sister)
A second pass, once per batch: it checks whether the compact snippet dropped important specifics (exact numbers, named plans/tactics, promises, capabilities, identities) and, if so, attaches a short detail note to that snippet — injected alongside the summary and never auto-edited. Empty for routine batches. Uses the same connection as the summarizer. View / edit / regenerate / delete each note in the Snippet Browser.
🧠 Character Ledger (psychologist)
A third background pass, once per batch: it maintains a living per-character model — Nature (stable personality & how they express themselves), Now (current volatile mood), Arc (slow relationship trajectory), and open threads (unresolved beats). Only the active cast (characters on screen in the recent window) is injected, so characters stay in-voice and evolve realistically instead of snapping out of character. Uses the same connection as the summarizer. View / delete entries in the Character Ledger panel above.

📌 Pinned Memories
Verbatim excerpts injected every turn inside <pinned> — immune to summarization, promotion, and branch repair. Pin via the button (uses your text selection, or the last message) or /sc-pin label.
🔎 Verbatim Recall
Two-phase lossless recall: an AI picks the relevant memory snippets for your query, then the ORIGINAL chat text behind them is fetched and injected for the next generation (ephemeral — auto-clears after it's used). Also via /sc-recall query. "→ Notepad" makes the last recall permanent.
🔗 Copilot Bridge (Author's Note)
Copilot's built-in Summaryception integration already feeds it your snippets automatically (Copilot-only, no duplication to your roleplay AI) — but it never reads your notepad or detail notes. These buttons copy just that missing part (notepad + details) into the Author's Note so Copilot sees your canon too, then let you pull it back out. Your existing Author's Note is stashed in a non-injected slot and returns on Restore. While mirrored, this small block is briefly also sent to your roleplay AI (harmless — it's your own canon), so Restore when you're done.
Requires Debug Mode to be enabled.
